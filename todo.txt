## Retune after fixing NNUE scale


  High Impact (margins scale linearly with eval)

  ┌──────────────────┬─────────────────────┬─────────────────────────────────────────────┐
  │    Threshold     │       Current       │                What it does                 │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ RFP margin       │ depth×85 / depth×60 │ Prune if eval exceeds beta by this much     │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ Futility         │ 100 + lmrDepth×100  │ Prune quiet moves when losing by this much  │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ Razoring         │ 400 + depth×100     │ Drop to QS when losing badly                │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ ProbCut          │ beta + 200          │ Skip positions winning by a lot             │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ NMP eval divisor │ /200                │ Extra null-move reduction per cp above beta │
  ├──────────────────┼─────────────────────┼─────────────────────────────────────────────┤
  │ Aspiration delta │ 15                  │ Initial search window width                 │
  └──────────────────┴─────────────────────┴─────────────────────────────────────────────┘

  Medium Impact (thresholds in cp space)

  ┌────────────────────┬────────────┬────────────────────────────────────┐
  │     Threshold      │  Current   │            What it does            │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ SEE capture        │ -depth×100 │ Prune bad captures                 │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ SEE quiet          │ -20×depth² │ Prune bad quiet moves              │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ Singular margin    │ depth×3    │ How much worse other moves must be │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ Eval instability   │ 200cp      │ Triggers wider QS SEE threshold    │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ Delta pruning (QS) │ +200       │ QS futility margin                 │
  ├────────────────────┼────────────┼────────────────────────────────────┤
  │ Contempt           │ 10         │ Draw avoidance penalty             │
  └────────────────────┴────────────┴────────────────────────────────────┘

  Time Management (in cp, medium impact)

  ┌──────────────────────┬──────────────┬───────────────────────────────────────┐
  │      Threshold       │   Current    │             What it does              │
  ├──────────────────────┼──────────────┼───────────────────────────────────────┤
  │ Score delta stable   │ ≤10cp        │ Reduce time when score barely changes │
  ├──────────────────────┼──────────────┼───────────────────────────────────────┤
  │ Score delta volatile │ >25cp, >50cp │ Extend time when score swings         │
  └──────────────────────┴──────────────┴───────────────────────────────────────┘

  Low Impact (not eval-dependent)

  LMR C=1.5, LMP 3+d², history -2000×depth, captHist -2000 — these are move-count or history-score based, not directly
  cp-dependent.

  That's roughly 12-15 thresholds to retune. The good news: they're all independent 1-line changes we can SPRT in
  parallel. The RFP, futility, and razoring margins are the most impactful — if the eval scale is ~3-5x larger with
  correct calibration, those margins need to scale proportionally.

#  Rules of thumb I find handy:

## elo

  - 100 Elo ≈ 2:1 odds — the most useful anchor point
  - 200 Elo ≈ 3:1 — a clearly dominant engine but the weaker one still wins occasionally
  - 400 Elo ≈ 10:1 odds — a full piece advantage in "engine class"
  - Every 70-80 Elo roughly doubles the win:loss ratio (ignoring draws)
  - Doubling time control is worth roughly 50-70 Elo for most engines
  - Doubling hash size is worth roughly 5-15 Elo (diminishing returns)

## Performance to ELO

Elo ≈ 70 * log2(speedup)

So a 2x increase in NPS is only worth 70 ELO.
A 10% performance increase is worth 10 ELO.

Given more data, and some Texel/NNUE training has often found us 50+ ELO, that's like a 70% speed increase.

## Self-play vs other engines

Search/movegen improvements in self play are only worth about 50% of that in matches against other engines. As against other engines they don't have the same eval blindpots

## Training Data

Effective information ≈ positions × sqrt(search_depth)

But value of positions decays after a while.
